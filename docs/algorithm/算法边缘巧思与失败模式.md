# Akasha：边缘 case 里的巧思，和它会怎么翻车

> 这是《[算法数据流讲解](算法数据流讲解.md)》的续篇。上一篇只讲了**成功面**——「绷带」联想出整段就医、「杀戮尖塔」联想出整张社交关系图。但一个会随时间生长的关联网络，真正有意思的部分都在边缘：它**什么时候会失灵**、**怎么不让自己烂掉**、**怎么忘**。
>
> 这篇把四件事讲透，每件都配一段**三维循环动画**（真实库脱敏数据，见文末）：
> 1. 右脑的两副面孔——同一套扩散，什么时候是涌现，什么时候被 hub 通吃
> 2. 自清洁——噪音边怎么被时间「饿死」
> 3. 四道 hub 抑制防线 + 冷启动 + 方向性 STDP
> 4. 短期抑制与遗忘——系统怎么「忘」

## 一、右脑的两副面孔：涌现，还是被 hub 通吃？

上一篇说右脑「顺着边把相似度排不上的记忆捞回来」。诚实地讲：**这件事有一半时候会失灵**，而且失灵和成功**长得几乎一模一样**——都是"高激活、一堆图扩散命中"。

差别不在扩散的**量**，在扩散的**落点**。

**成功面（真涌现）**：query 落在一个**强自连簇**上。
- 「那还要搞绷带吗」——只有"绷带"二字、毫无医学语义，但它命中的种子身上挂着整段就医期天天一起说的强边，能量顺边流回**麻药·引流条·换药**，全是 dense 给不出的补充。这是右脑存在的理由。

**失败面（hub 漂移）**：query **没有语义锚点**（"我现在状态怎么样"、"现在呢"、"讲"），或一句话塞了**多个话题**。
- 没有锚点，能量无处可去，就被全库**连接度最高的几个 mega-hub**（天气、"说说看我都有什么仓库"、问候关系簇）吸走——它们跟任何 query 都一两跳可达。
- 结果：top 命中是 fan 100~400 的超级 hub，激活值很高，却**和你问的毫无关系**。

> **一句话：高激活 ≠ 有推理。** 激活值再高，也可能只是 hub 的引力，不是联想。决定成败的是 query 有没有落在一个**强到能赢过全局 hub 的本地簇**上。

这不是 bug，是 spreading activation 的**结构宿命**：能量总会往连接最密的地方流。系统用四道防线压制它（见第三节），但压不死——所以**无锚、跨域、纯问候的 query，右脑基本是在帮倒忙**，这类应当短路图扩散、回退 dense。

🎬 对照动画：`emergence_bandage`（真涌现）vs `hubdrift_status`（被天气 hub 通吃）——同一套扩散，两种命运。

## 二、自清洁：噪音边怎么被时间「饿死」

日常聊天里全是**物理巧合**：聊"拉肚子"时随口插一句"快递放消防栓上了"，STDP 会忠实地在它们之间连一条边（它不预设语义，只信时间上的共现）。这种噪音边怎么处理？

答案是：**不删除，饿死它。**

机制只有一条朴素的定律——**边权按 14 天常数做 lazy 时间衰减**（`core.py:542 effective_edge_weight`）：

```
有效边权 = 原始边权 × exp(−闲置时长 / 14天)
```

关键在于这个衰减是**惰性、按需**算的，配合「用进废退」会拉开惊人的差距。看一组**真实库的数字**（同一个节点的两条出边）：

| 出边 | 原始边权 | 闲置 | 有效边权 |
|---|---:|---:|---:|
| → 最近还在一起出现的话题 | 0.36 | 0.6 天 | **0.348** |
| → 「昨晚睡得怎么样」（曾经天天聊） | **0.61** | 38 天 | **0.041** |

第二条**原始强度更高**（0.61 > 0.36），却因为 38 天没被再次共激活，有效权重掉到 1/8。而被反复唤起的边——每次召回后 `bounded_add` 再加固一点（`core.py:550`，越接近 cap=2.0 长得越慢）——稳稳保持。**赢者通吃**就这么发生：真实回路越走越粗，噪音边在对比下萎缩成「背景辐射」。

这里有个常被误解的点，必须钉死：

> **Hebbian 学习不负责"删错误"，它只负责"疯狂奖励反复共现的真实关联"。** 噪音节点和那条细弱游丝**永远躺在图里**（保留生物学痕迹），但功能上被彻底静音——能量到了岔路口，绝大部分顺着粗边走，只有一丝流向噪音，再加上它字面也不像，最终得分惨淡，跌出注入名单。

为什么这能成立、而不会精神分裂？因为还有**全脑融合截断**兜底（见上一篇 seed_pool）：dense（左脑·字面常识）永远把最稳固的事实锚在候选池头部，噪音因为缺 dense 兜底，连**强化池都进不去**，于是连「被再次加固」的资格都没有——彻底断了死亡螺旋。

🎬 对照动画：`pruning_decay`——同一节点的出边，初始全亮（raw 强度差不多），随时间播放，没复习的一条条变细断开，只剩被反复共激活的骨架。

## 三、四道 hub 抑制防线（+ 冷启动 + 方向性 STDP）

hub（谁都连的高 fan 节点）是这套系统最大的敌人（见第一节）。它不是靠一个参数压住的，而是**四道独立防线、各管一段**，纵深防御：

| # | 位置 | 公式 | 管什么 |
|---|---|---|---|
| 1 | 种子初始能量 | `e0 = 能量 / √(1+fan)`（`core.py:925`） | 出发点就给 hub 打折 |
| 2 | 节点状态权重 | `state = exp(…)·resource / √(1+fan)`（`core.py:853`） | 扩散过程里 hub 易激活度被压 |
| 3 | 转移矩阵 | 每列只留**最强 12 条边**再归一化（`core.py:899/907`） | hub 的弱边被裁掉，能量只走铁关系 |
| 4 | 一跳电导归一 | `边权 / √(出度强度·入度强度)`（`core.py:1145`） | "谁都连"的 hub 被分母除小，只有**专一**的邻居冒头 |
| + | 最终打分 | `/ (1+fan)^0.1`（`core.py:998`） | 收尾再压一道 |

为什么要四道？因为 hub 会在**每一个环节**重新冒头：你在种子端压了它，它在扩散里又靠高 fan 涨回来；你在扩散里压了它，它在打分时又靠路径数涨回来。**单点防御必被绕过**，所以每一层都得独立设防。即便如此，第一节那种无锚 query 仍会翻车——说明 hub 抑制是个**永远做不完的攻防**，目前的结论是：与其调参，不如**识别无锚 query 直接短路图扩散**。

另外两个边缘巧思，顺带钉死：

**冷启动：第一个 query 必然零召回。** 空库进来，`if not nodes: return [],[],trace(0,0)`（`core.py:1234`）——检索给 LLM 的记忆是空的。系统第一步只「长出一个孤立节点」，**边要到第 2 个 query 才出现**（得先有旧节点被激活，才能与当前 turn 共激活连边）。而且**学习滞后一轮**：检索用的永远是**上一轮 commit 时**的图，本轮的边和状态更新发生在回答**之后**。所以 Akasha 永远在用「昨天的图」回答「今天的问题」。

**方向性 STDP：边是有向的、不对称的。** 召回之后建边时（`core.py:282`），因果边（旧记忆 → 当前轮）权重 **1.0**，反因果边（当前轮 → 旧记忆）只有 **0.35**（`STDP_ACAUSAL_EDGE_GAIN`）。这把「时间箭头」编码进了图结构——「先发生的更可能引发后发生的」，而不是反过来。真实库里实测因果边 ≈ 反因果边的 2.8 倍，与这个比值吻合。

## 四、短期抑制与遗忘：系统怎么「忘」

Akasha 有**三条时间常数**同时在跑，各管一种「忘」：

- **strength 长期强度 · 7 天**：编码即峰值，之后指数衰减（Ebbinghaus 遗忘曲线）。
- **resource 短期资源 · 30 分钟**：被取用就暂降，慢慢恢复（短时突触抑制）。
- **edge 共现边 · 14 天**：见第二节。

**短期抑制**最反直觉、也最实用：每次召回后，命中节点的 resource 被扣（`core.py:1346`，`resource *= 1 − 0.35·min(1,score)`），30 分钟才按 `1−(1−r)·exp(−Δ/30min)` 恢复（`core.py:530`）。后果是——

> **同一句话在几分钟内连发，召回会一次比一次弱。**

真实库实测：「我现在健康状态怎么样呢」几分钟内连发三次，最强命中节点的 score **3.73 → 1.57 → 1.46**，resource **1.00 → 0.69 → 0.66**。这不是 bug，是**防短期刷屏**：不让同一批记忆因为你反复问同一句就垄断召回；隔半小时再问，resource 恢复，它们又回来了。

**遗忘的另一面：淡忘的记忆，靠强边能被唤回。** strength 会随时间衰减到很低，但**节点不删**。上一篇 limboo 例里「镜之边缘」长期强度已衰减到归一化约 0.2、cos 才 0.44，却照样被召回——**全靠那条共现边够强**。所以：**边不只盖过"不像"，也盖过"忘了"**。

🎬 对照动画：`resource_depression`——同一句连发三次，被召回的节点逐次变暗，第四帧（+30min）恢复亮起。

### 闭环一句话（接上一篇）

> 上一篇说 Akasha 会「联想」；这一篇说它**也会失灵、会自清洁、会忘**——而正是这三件「不完美」，让一个会生长的关联网络**不至于烂成一团 hub**。真正的鲁棒不在召回更多，而在**知道什么时候不该联想、让没用的自然淡出**。
