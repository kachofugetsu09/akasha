# 召回对照报告 · 线上库 base vs heterosynaptic-only

> **base**=生产快照。**hetero**=唯一改动:在原算法基础上加 **heterosynaptic 压抑**
> (Chistiakova & Volgushev 2013):节点本轮被强化时,其余非活动出边各自 ×(1−β=0.05),Δw∝w。
> 补上原算法缺失的 Hebbian 稳态对偶项(纯 homosynaptic→必然 runaway→hub)。
> 曾试过 ACT-R fan-effect,已撤——蒙特卡洛零模型证明'junk vs 有意义'不是图结构属性,
> fan 惩罚会替用户做价值判断且误伤情绪主线;heterosynaptic 只'按使用判价值'(用进废退),不替用户判断。

> `跨`=召回节点 session≠query;lag=与 query 出生时间差(天,同session)。

> 注:heterosynaptic 在 edge 层面强(hub 出边权重和 71→16),但召回层面变化温和——
> 因 Graph lane 的 `edge_signal=eff_w/√(out·in)` 已按总强度归一,对整体边权缩放近似免疫(∝√k)。
> 这是诚实的局限:它治住了边权 runaway,但 hub 在召回里的霸占只被温和削弱。
