## 引言 ✅（`introduction.md`）
两句 slogan（messages, not memories / recommendation, not retrieval）+ 批判现有系统两病灶（模仿检索·幻觉链；不定义"重要"只能拿朴素代理凑数）。结尾抛问：什么才算重要？凭什么记、凭什么忘？

## 一句话桥 + 两种回忆 ✅
- 答案先亮：重要性是召回时由结构算出来的；下文自底向上搭结构。
- **两种回忆（dense/ripple）概念定义**——slogan 的落地，全文主轴对偶：
  - Dense（左脑·相似检索）= retrieval、地板、无状态不走边。
  - Ripple（右脑·涟漪扩散激活，RAR）= recommendation、涌现、走边找"不相似却相关"。
  - limboo 一句话对照。只讲"是什么/为什么要两种"，机制留 Part 4。

## Part 1 · 消息即唯一真相：到底存什么 ✅
- 节点 = 指回原始消息的指针（不复制不提炼）→ message as only truth。
- 三召回底座：消息级/turn 级 embedding、FTS+`fts_token_idf`、全局重心 `akasha_salience_state`。
- 三状态变量 `salience`/`strength`/`resource` + 初始化公式（σ、strength₀、resource=1）。
- 刻意克制：**编码不判重要**（对打 count/recency/LLM 代理）。
- 钩子→ 孤立的灯，要靠连边才"重要"。

## Part 2 · 边：关联用进废退长出来 ✅
- 读写分离解耦合（检索=读、学习=回答后异步滞后一轮写）。
- 边=有向带权（`weight`/`co_count`/`last_used_ts`），记"共现"非"相似"。
- STDP 规则：因果 1.0 / 反因果 0.35 / 共激活 √(sᵢsⱼ)；**非对称三层讲透**（STDP 借鉴、对称会抹平时间箭头、0.35≈人类前向不对称/TCM）。
- 落库：`bounded_add` 饱和封顶 2.0；**惰性按需衰减**（〈值,时间戳〉、读时现算、O(被碰到)、指数可叠=精确）；三条衰减共用此机制。
- 冷启动：第一条边靠稠密相似度引出。
- 钩子→ 边会枯荣、节点也在时间里被筛，且这套时间机器**拒绝"次数=重要"**。

## Part 3 · 时间动力学：忘，是为了留住重要的 ✅
**定位**：节点状态随时间怎么变；正面回答引言的"凭什么忘/凭什么留"，并证明**它不是数次数**。
- 复用 Part 2 的惰性衰减机制（不再重讲）。
- **接 Part 2 的钩子**：节点侧强化 = 这一部分的入口。`strength ← bounded_add(衰减后, 0.18·s, 3.0)`（LTP），resource 扣、recall+1。
- `strength`：编码即峰值 → 7 天指数衰减；被召回时 `bounded_add` 抬升（复习巩固，LTP）。
- `resource`：短期抑制——被取用即扣、30 分钟指数恢复；防短期刷屏（同句连发召回逐次变弱）。
- **核心硬正样本**：`corr(strength, recall)=−0.23`、`corr(strength, fan)=−0.36`——越被频繁召回/越是 hub，strength 反而越低。**系统在结果上拒绝了"使用次数=重要"**（兑现引言对 count 的批判）。配「你好啊 recall=101 → str=0.10」对照。
- 例子：手指 episode 的节点随天数 strength 怎么走 vs 寒暄怎么沉底。
- 钩子→ 状态和结构都备好了；到召回那一刻，怎么把它们读成"此刻什么重要"。

## Part 4 · 召回 = 推荐：重要性在这一刻被算出来 ✅（最重的一部分）
**定位**：兑现 slogan 2；这里才是"什么是重要"的落点——不是查询，是推荐。
- 种子 seed：Dense(cos>0.675) / FTS(稀有词) / BlackHole(高显著直连) 三路（limboo 命中 Dense+FTS）。
- **图扩散：graph_expand 主力发现，RWR 辅助微调**（非平行通道）：
  - graph_expand 管发现：整图一跳、**无时间限制**、电导归一 `w/√(出·入)`、独立打分——64.5% 激活 items 来源，涌现主力（limboo 宝物 + 鱼石脂全在此）。
  - RWR 管微调：30min 微图内能量再分配——只在 direct（cos）接近时当 tiebreaker，不发现新节点，拿掉系统仍可工作。
  - Bridge：2hop 桥接，184/3352 条 query 中极少出现，且 85% dense 本来就能找到——**一笔带过**。
  - **为什么 graph_expand 必须独立存在**（而不是放大 RWR 窗口）：① N² 计算成本（60² vs 3000²）② hub 虹吸（窗口越大噪音边越多，hub 通吃）③ 数学逻辑不同（RWR=零和竞争适合稠密微图，graph_expand=独立打分适合稀疏整图）。
- 打分：noisy-OR 融合（ripple=current·3·state, direct=0.5cos）× 三状态增益 ×（hop/user/fan 惩罚），阈值 0.22。
- **五道反 hub 防线**汇总（√(1+fan) ×2、列 top12、电导归一、(1+fan)^0.1）。
- **limboo 杀招**：dense 第 22/24/1199 名、cos 0.44–0.60 全够不到，靠 graph_expand score 2.40 登顶 = 模式补全。
- **鱼石脂杀招**（新增）：cos=0.522 不在 dense top-10，离种子几小时远不在微图，graph_expand 独自发现+打分，ripple=1.252 score=3.09 = dense #1 的 3.9×。
- **三集合理清（混乱源头）**：`dense_items`（纯 cos top-K，注入·精确）/ `ripple_items`（扩散召回去掉 dense 已给的，注入·纯增量）/ `activation_items`（左右脑融合 top-8，**写回**用）。
- 注入：左脑(精确)/右脑(纯增量) 分块。
- 钩子→ 这次召回的融合集，回答后会被写回成边。

## Part 5 · 闭环：今天的召回，织成明天的图 ✅
**定位**：把"读"(Part4)和"写"(Part2 学习规则)接成环，点明它是活的。
- 召回结束 → `activation_edge_updates` 对 `activation_items` 落 STDP 边 + 命中节点 strength↑/resource↓/recall+1（Part 2/3 的规则在此触发）。
- **全脑融合强化**（源码已核：写回吃 `activation_items`=左右脑融合 top-8，engine:560/623，不分左右）：dense 事实和 ripple 联想织进同一套边；且 dense 占强化池头部 → 纯右脑挖出、无 dense 兜底的噪音挤不进 top-8 → 不被强化 → **防右脑回音室/死亡螺旋**。
- 滞后一轮、异步：永远用上一轮的图回答这一轮。
- 涌现综述：**重要性是怎么"拼"出来的**——编码给弱起点 → 边给关联 → 时间筛存活 → 召回按语境现算。没有哪一层单独是"重要"。
- 一句话闭环：常一起想起的连得越紧、常用的留得越久、不用的淡出；普通向量库是快照，Akasha 随你生长。

## Part 6 · 实验与诚实的边界 ✅
**定位**：把账摊开，正样本+失败模式都给。
- **salience 饱和实测**（中位 0.87、对最终重要度零相关）→ 反过来支撑"重要性不该在编码定"；标准化/NN/TCM 三种去饱和对照（结论：novelty≠importance）。
- **右脑两副面孔**：簇强→真涌现（limboo/绷带）；无锚/跨域→被 mega-hub 通吃（失败面）。好坏由语义落点决定，不由扩散量。
- "拒绝次数=重要"的负相关（呼应 Part 3）。
- 三维循环动画 case（`private/viz/dist/`）作可视化证据。
- 局限：冷启动零召回、桥触发少、affect 簇需限流、实时事实需门控。
