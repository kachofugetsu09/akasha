# 引言

面向个人 agent 的记忆系统，现在全走错了路。

病根只有一个：**它们做的是检索系统，不是记忆系统。** 别的毛病，都是从这一个长出来的。

## 一、用问答的尺子，量错了东西
看 benchmark 就够了：清一色在比 which / how / what。把记忆当问答 bot 来考——从根上就烂了。
可个人 agent 里，人对它说的是陈述句、命令句，不是在出题。用户要的不是"我问你才答"，是你**顺着当下聊的，主动补全你对我这个人的印象**。一个只会躺着等检索的系统，从第一天起就答错了题。
更致命的是过分依赖LLM导致的幻觉。LLM 回一次话，一层幻觉；把对话提炼成记忆条目，又一层；检索时改写查询，再一层。提炼得越勤，幻觉叠得越厚。这种系统只会抠 fact，不会补全上下文，还动不动抠错——它一旦把错的东西塞给你，你当场就觉得恶心：它本该懂你，却用错了你。这口恶心，会让人从根上不再信任这个 agent。

## 二、连"重要"都没定义，谈什么遗忘
多数系统干脆无状态；少数学着做遗忘的，是照猫画虎——根本没想过为什么要有状态，状态只是拿来装样子。
遗忘的本质，是**为了记住重要的**。可什么叫重要？**没人定义。** 不定义重要，遗忘就是为删而删。
于是它们抓几个朴素代理来冒充"重要"，每一个都荒唐：
- **使用次数**——量的是热门，不是重要。而个人场景里高频的全是噪音（晚安、查天气、在吗），真正要紧的事一辈子可能只说一次（我做了手术、他是我舍友）。按次数记忆，就是**专记废话、专忘要事**，跟遗忘的目的反着来。
- **最近用过**——把"刚发生"当"重要"。于是记忆被近期琐事刷屏，昨天的大事被今天一句"在吗"顶掉。
- **让 LLM 打分**——再叠一层幻觉；而且它判的是"看起来重要"，不是"对你重要"。
三个代理，一个病根：它们都把重要性当成一个**事后才数、向后才看、没有语义**的标量。数次数、比新旧、让模型猜——没有一个在回答那句真问题：**这件事，对这个人，到底值不值得记。**

所以 Akasha 在这里定义新一代的记忆系统范式，他的核心就是两句话：

the message is the only truth.
to remember is to recommend.

消息即真相，记忆即推荐。



> **这个算法的核心，不是一个公式，是一条关于"重要性"的原理：**
>
> **重要性不是写入时贴上去的标签，而是租来的——靠一条记忆"持续被需要"来续租；而遗忘，就是交不起租金时的到期。**
>
> 几乎所有记忆系统都在**写入那一刻**就给每条记忆判一个重要性：用使用次数、用 recency、用 LLM 打分。判得太早、太自信，还把当时的幻觉一起固化进库——一旦贴错，错就永远留着。Akasha 反过来：**写入时一律给中性起点、不判重要**；一条记忆到底重不重要，留给它在后续每一次召回里**重新去挣**。挣得到（反复被真正需要）就在使用中自我加固；挣不到（没人再想起）就被时间慢慢饿掉。
>
> 这条原理之所以能成立，是因为整个系统由**一个量**驱动、且这个量同时管"读"和"写"：每次召回那一刻算出的 `score`（这条记忆此刻有多相关），**既决定注入什么给 LLM（读），又决定哪些节点和边被加固多少（写）**。判断"此刻什么相关"的那个数，就是学习信号本身。于是——
>
> - **常被真正需要的记忆**：score 高 → 被加固 → 下次更易被想起 → 越滚越强（有上限，不会爆）；
> - **无关的巧合记忆**：score 低或再不出现 → 不被加固 → 被衰减饿死、自动静音。
>
> "好的复利"和"坏的自净化"不是两套机制，是**同一条租金原理的两个方向**。没人单独去写"奖励重要的"或"清理噪音的"——把"相关性即学习信号 + 万物衰减 + 有界增长 + 有底兜底"这几块彼此制衡的规则拼对，剩下的自组织、自净化、自复利全是**涌现**出来的。这套东西更像一个寻找平衡态的**活的动力系统**，而不是一个被查询的数据库。
>
> **召回那一刻，两路证据如何融合成 score。** 一路是**向量相似度**（direct · 左脑 · 检索，静态不变），一路是**图扩散能量**（ripple · 右脑 · 联想，动态会学习）。它们被一行 noisy-OR 融合：
> $$\text{content\_base} = \text{strong} + \text{weak}\cdot(1-\text{strong})$$
> 哪一路强都能把它拉高，两路都强趋近 1.0。这行公式只是**结算机制**——真正的灵魂是上面那条租金原理，noisy-OR 不过是它在一次具体召回里的算账方式。
>
> - **Direct（左脑）**：query 和每条记忆算 cos，无状态、不走边，就是最近邻。所有向量库都这么干。它是**地板**——永远兜底，保证准确性。
> - **Ripple（右脑）**：把线索当种子丢进关联图，能量顺着"过去一起被想起"的边外扩，点亮那些**不相似、却相关**的记忆。这是稠密相似度给不出的"联想/推荐"，是 Akasha 真正不同于普通向量库的地方。（引擎名 **RAR = _Ripple Activation & Recall_**。）
>
> 一句话对照：你说"和 limboo 玩杀戮尖塔"，**dense 还你"杀戮尖塔"的各种说法；ripple 还你"亡灵契约师 / 舍友联机 / 都玩机器人"——一整段关系。**
>
> 下文的主干就是这条租金原理如何被实现、又如何自我维持：编码先给一个**不判重要的中性起点**（一、消息即真相）；边和时间是"租金"的两套实现——关联用进废退、状态用则留不用则忘（二、边；三、时间动力学）；召回那一刻把所有零件结算成 score（四、召回=此刻结算）；最后这个 score 写回去，让今天的召回织成明天的图、并在长期对话里和你共同演化（五、闭环）。末了诚实地说清它的家在哪、边界在哪（六）。

## 一、消息即唯一真相：记忆系统到底存什么，以及为什么"先不判重要"

Akasha 不提炼记忆。一轮对话——你说的话，加 agent 紧接着的回应——对应一个**节点（node）**；但节点存的不是消息的副本或摘要，而是一个**指回原始消息的指针**（key 形如 `{会话}:N`）。原始消息原样留在宿主库里，是唯一的真相源；我们一个字都不归纳、不重写，只在它**之上建索引**。这是最核心的设计：**消息即唯一真相**——因为每多一层提炼就多一层幻觉，而唯一不会在转述中变形的，是用户真正说过的那句话。这也呼应了开篇的租金原理：连"内容"都不敢替你定型，更不会替你预判"重要性"。

在这份真相之上，写入时我们顺手备好三种**召回底座**，全都只是索引、不动原文：**向量**——原始消息逐条嵌入存进 `akasha_embedding_cache`（消息级），节点再存一份这一轮的加权均值向量（turn 级），两套粒度让稠密召回在更细的消息级匹配、再映射回节点；**字面**——对原始消息建 FTS 索引，配一张 IDF 表（`fts_token_idf`）滤掉高频废词，专门接稀有专名（比如 "limboo"，embedding 对生僻词不稳，FTS 按字面一抓一个准）；**新颖度基线**——一个增量维护的**全局重心** $c$（`akasha_salience_state`），即"你至今说过的一切"的平均方向。

然后，每个节点再挂三个会随时间变化的**状态变量**：`salience`（新颖度，相对全局重心的意外性先验）、`strength`（长期强度，一个会随时间衰减的峰值）、`resource`（短期资源，一份防刷屏的预算）。先把它们的身份钉死——**它们是后续机制的输入零件，本身都不是"重要性"**；把任何一个单拎出来，都不等于一条记忆重不重要。重要性是这三者加上边、加上语境，在召回那一刻被"算"出来的。

具体怎么算。`user` 消息（序号 N）和紧跟的 `assistant` 回应（N+1）合并到同一节点（回应归到它前面的发问轮）；节点向量是两条向量按条数的加权平均。三个状态变量在写入那一刻这样**初始化**（随时间怎么演化留到第三部分）。`salience` $\sigma$ 拿这句的单位向量 $v$ 和全局重心 $c$ 作比：

$$\sigma=\operatorname{clamp}\big((1-\cos(v,c))\times 2,\;0,\;1\big),\qquad c=\operatorname{normalize}\!\Big(\sum_{i\,<\,t}\hat v_i\Big)$$

注意 $c$ **只累加此前的消息**（因果，不看未来）——越偏离你说过的一切，$\sigma$ 越高；第一条消息没有"过去"，$\sigma=0$。一个节点取 `user`、`assistant` 两条里较大的 $\sigma$。`strength` 则是"编码即峰值，越新颖起步越牢"：$\texttt{strength}_0 = 3.0\times(0.70+0.30\,\sigma)\in[2.1,\,3.0]$。`resource` 满格 $1.0$。

这里有一处**刻意的克制**，也是整条租金原理的起点：**编码阶段，我们不判断一条记忆重不重要。** 用次数的、用 recency 的、用 LLM 打分的系统，都在写入那一刻就急着贴标签，判得太早、还把幻觉固化进库。我们反其道而行：编码只负责**保真**和**给一个中性起点**；一条记忆到底重不重要，留给它后来在关联结构里的位置、在时间里的存活、在某次召回时的语境去**挣**。

举一个会贯穿全文的例子。某天你说：「肿得厉害，去医院切开引流，塞了根引流条。」这轮被原样编码成一个节点，挂上一组中性初始状态。就在此刻，它和库里其他几千个节点没有任何分别——是一盏**孤立的、还谈不上重不重要的灯**。要让它"重要"起来，第一步是看它如何与别的记忆连成一片。那是下一部分的事。

## 二、边：关联是"用进废退"长出来的——租金原理在"关系"上的实现

检索是**读**：用现有的图算出召回。学习是**写**：在你这轮回答之后异步发生，且**滞后一轮**（这轮写下的边，下一轮才用得上）。所以讲边，我们只需把"这次召回浮现了哪些记忆、各自多强"当作已知输入（怎么选出来——第四部分），专注在召回之后、边怎么被写下来的那条学习规则上。

**一条边是什么。** 有向、带权（`weight`，另记 `co_count` 共现次数、`last_used_ts` 上次共激活时刻）：`A → T` 的权重，记录的是"过去 A 和 T 被一起想起过"的强度——注意，**不是 A 和 T 相似，而是它们共同出现过**。这正是稠密相似度给不出的那一维。

**学习规则（Hebbian / STDP）。** 设你这轮 `T` 的召回，按 score 排名只取**8 条**旧记忆 $\{A,B,\dots\}$（`activate_limit=8`，这 8 条就是"激活集"——强化只在它们身上发生，第 9 名起连被强化的资格都没有，这本身是第一道"强者通吃"的筛）。这个 8 是激活集的容量上限，和人的工作记忆容量（7±2）一个量级。回答之后，落三种边：**因果边** $A\to T$ 增量 $\propto s_A$（增益 `STDP_CAUSAL=1.0`，旧记忆"引发"了当前轮）；**反因果边** $T\to A$ 增量 $\propto 0.35\,s_A$（`STDP_ACAUSAL=0.35`，当前轮"回指"旧记忆，但只有因果的 0.35）；**共激活边** $A\leftrightarrow B$ 增量 $=\sqrt{s_A\cdot s_B}$（同一次回忆里一起被点亮的两条，按几何均值互连）。

那个 **$1.0$ vs $0.35$ 的不对称，是这条规则的灵魂**——它让图学到的是**有方向的预测/因果结构**，而不是无向的"相关"。借的是神经突触的 **STDP（脉冲时序依赖可塑性，Bi & Poo 1998）**：若 A 在 B *之前* 放电（A 可能"引发"了 B），突触 $A\to B$ 被增强；反过来则被压制。它给经典 Hebb 的"一起放电就连上"补上了**时间方向**——连的不是"谁和谁同时出现"，而是"谁倾向于引出谁"。

如果对称，边就退化成纯共现，时间箭头被抹平：系统再也分不清「拉肚子 → 吃药」和「吃药 → 拉肚子」，召回时能量前后等流，**后发生的事会以满强度反向"预测"先发生的事**——你丢掉的，正是 episode 之所以成其为 episode 的那条线：顺序。而 $0.35$ 也不是凭空拍的：人类自由回忆里有个稳健现象——**前向不对称（temporal contiguity effect，Kahana）**，想起第 $i$ 项后更容易接着想起 $i{+}1$ 而非 $i{-}1$，反向/正向之比大致就在这量级。Akasha 的 $0.35$ 落在这区间里——保留回指能力（不为 0），但把前向定为主干。

**怎么落库（用进废退）。** 这是租金原理在边上的具体形状。库里存的从来不是"当前值"，而是一对〈权重, 上次更新时刻〉。强化时，**先把这段时间的衰减结清、再加增量、再把时间戳拨到现在**——增量先乘 $0.12$，并做**有界增长**（越粗长得越慢，封顶 $2.0$）：

$$w \leftarrow \operatorname{bounded\_add}\big(\underbrace{w\cdot e^{-(t_{\text{now}}-t_{\text{last}})/14\text{天}}}_{\text{先结清衰减}},\; 0.12\cdot\Delta,\; 2.0\big),\quad \operatorname{bounded\_add}(v,\delta,c)=v+\delta\big(1-\tfrac{v}{c}\big)$$

这里的 $\Delta$ 就是上面三种边各自的增量。拿一条因果边（$s=1.0$，$\Delta=1.0$）走一遍，比公式直观。**第一次共激活**，边从零开始：$0.12\times 1.0\times(1-0/2.0)=0.12$。**3 天后第二次**，先结清 3 天衰减 $0.12\times e^{-3/14}\approx 0.10$，再叠增量 $0.12\times(1-0.10/2.0)\approx 0.11$，写到 0.21。**又过 10 天第三次**，衰减到 $0.21\times e^{-10/14}\approx 0.10$，再叠 0.11，仍写到 0.21——10 天的衰减正好吃掉了上次的增长。这条边想继续变粗，要么共激活更频繁（间隔远小于 14 天），要么 score 更高；如果就此不再一起出现，14 天后跌回 0.08，功能上静音。这就是"用进废退"的精确形状，也是**租金**：每共激活一次就在衰减后的旧权上再叠一阶，$1-w/2.0$ 让越接近上限涨得越少——反复一起想起的边一阶一阶往 $2.0$ 爬，久不复习的边自己枯掉。一次召回最多写 $8+8+\binom{8}{2}\times 2 = 72$ 条边。（强化还有节点侧的一半：这 8 个被点亮的节点本身也被强化——`strength` 涨、`resource` 扣、`recall_count+1`，那是节点的 LTP，第三部分讲。）

**衰减是"惰性、按需"算的。** 没人碰它时，一条边连同那个旧时间戳原封不动躺着；只有当它在某次查询里真的被读到、或被再次强化时，才现算它到此刻的有效权 $w_{\text{有效}}=w\cdot e^{-(t_{\text{now}}-t_{\text{last}})/14\text{天}}$。全库三千多节点、十几万条边，从不需要被逐个扫衰减——每次查询只对碰到的那几十个现算，代价是 $O(\text{被碰到})$ 而非 $O(\text{库})$。而且指数可叠，惰性地一次算完 30 天衰减，和每天扫一次连算 30 次结果完全相同——既省又精确。这套〈值, 时间戳〉+ 读时现算是通用的：边的 `weight`（14 天）、节点的 `strength`（7 天）、`resource`（30 分钟）三条衰减全共用它。

**第一条边从哪来？** 冷启动时图是空的：第一轮召回什么都没有，写不出边。但召回不只靠边——稠密相似度不需要边就能把相似的旧轮捞回来。于是第二轮起，相似度先把旧轮共召回，写下第一条边；此后越用越密。limboo 那张关系图、手指那段就医链上的粗边，全是这样在几十次共同提起里一点点长出来的——**不是写入时连的，是"用"出来的**。

到这里，我们有了一张会生长的关联图。但边只是把节点连起来；它会随时间枯荣，节点本身也在时间里被筛——而且这套时间机器，恰恰**拒绝了"次数=重要"**。

## 三、时间动力学：忘，是为了留住重要的——以及"重要性是租来的"的硬证据

先接上文那半句话：强化不只在边上发生，被点亮的节点本身也被强化。拿「切开引流」这个节点走一遍。假设它上次被召回是 3 天前，当前 strength = 2.0，resource = 1.0，这次又被命中、score = 2.0。更新分两步——**先结清 3 天遗忘**：7 天常数衰减，strength 先掉到 $2.0\times e^{-3/7}\approx 1.30$；**再叠这次巩固**：增量 $0.18\times\text{score}=0.36$，按有界增长打折（越接近上限 3.0 涨得越慢），实际加 $0.36\times(1-1.30/3.0)\approx 0.20$，strength 写到 1.50。resource 侧：这次 score 够高，消耗拉满 $\text{resource}\leftarrow 1.0\times(1-0.35\times 1)=0.65$，接下来半小时内再被召回排名会被往下压。

换个东西——「你好啊」。3 天前 strength 就剩 0.15，这次只是被 hub 引力扫到、score 才 0.5。衰减后 $0.15\times e^{-3/7}\approx 0.10$，增量 $0.18\times 0.5=0.09$，有界增长几乎不打折（离 3.0 太远），实际加 0.09，最后写到 0.19。**两条路线的差距就在这个 score 上**：「切开引流」被召回是因为 query 真的和它相关——content_base 高、score 高，一次加 0.20；「你好啊」被扫到是因为 fan 大、边杂——content_base 低、score 低，一次只加 0.09。**同样的"被想起一次"，真正被需要的加得多，被顺带扫到的加得少。** 这就是节点的 LTP：不是"被想起就强化"，而是"被需要才强化"——score 是那个筛子，也是租金的标尺。

上面只看了节点侧的 strength 和 resource，再加上 Part 2 已讲透的 edge 衰减——三条时间常数同时在跑，各管一种"忘"：

| 衰减 | 常数 | 管什么 | 被什么对抗 |
|---|---|---|---|
| `strength` | 7 天 | 这条记忆本身还记不记得 | 被召回时 `bounded_add` 抬升（复习巩固） |
| `resource` | 30 分钟 | 刚用过，暂时压一压 | 时间自然恢复 |
| `edge` | 14 天 | 这两个话题还常不常一起出现 | 被共激活时 `bounded_add` 加固 |

**为什么需要三条，而不是一条统一的遗忘曲线？** 因为它们管的是三个正交维度。一条记忆可以"记得很牢但暂时累了"（strength 高、resource 低——刚用过，过半小时再问又能满血）；也可以"淡忘了但关联还在"（strength 低、edge 高——会被强边救回）；也可以"关联断了但本身还记得"（edge 几乎没有、strength 还够——孤立冷门记忆，dense 还能捞到）。三条合在一起，让系统能区分"忘了"和"累了"和"不相关了"，而不是混成一个标量。其中 resource 的 30 分钟恢复效果是出乎意料的——真实库实测（seq 138/140/142，同一句"我现在健康状态怎么样呢"几分钟内连发）：

| 第几次 | 间隔 | 顶部节点 score | resource |
|---|---:|---:|---:|
| 1 | — | 3.73 | 1.00 |
| 2 | +204s | 1.57 | 0.69 |
| 3 | +28s | 1.46 | 0.66 |

同一批记忆一次比一次弱；隔 30 分钟 resource 恢复，又满血回来。防治了短期刷屏：不让同一批记忆因为反复问同一句就垄断召回。

** 引言批判了现有系统拿"使用次数"当重要性代理。线上库 3337 条节点状态统计：

$$\operatorname{corr}(\text{strength}, \text{recall\_count}) = -0.23,\qquad \operatorname{corr}(\text{strength}, \text{fan}) = -0.36$$

**越被频繁召回、越是 hub，strength 反而越低，这是机制设计的必然。strength 的增长靠 `bounded_add(衰减后, 0.18·score, 3.0)`，关键在那个 score——它不是"被扫到的次数"，而是召回时算出的内容基底 × 状态增益 × 惩罚链（Part 4 讲全貌）。一个 hub 节点（"你好啊"）虽频繁被扫，但每次都是因为它 fan 高、边广，不是 query 真和它相关——content_base 低、每次加得少；加上它 salience 低（不偏离重心），起点就低。低起点 + 低增量 × 频繁 = 还是低。反过来两个真实节点：

| 节点 | recall | fan | strength | 为什么 |
|---|---|---|---|---|
| 「你好啊」 | 101 | 174 | 0.10 | 每次都是被 hub 引力扫到的，不是真正被"需要"的 |
| 「肿得厉害，去医院切开引流，塞了根引流条」 | 4 | 28 | 2.31 | salience 高（偏离重心）起步就牢；每次被召回都因和当前语境真正相关 |

「你好啊」被召回 101 次 strength 快跌没了，「切开引流」只被召回 4 次 strength 却稳在 2.3——**"重要性"和"被用了多少次"在这套机制里是负相关的**。这正是"租来的，不是买断的"：交不起租（每次 score 都没内容支撑）的频繁项，照样留不住。

而且，**衰减到 0 的节点不删**。strength 跌到 0.10 的节点仍在库里，dense 相似度仍能捞到它（只要 cos 够高），但它对扩散的贡献几乎为零——退出了"主动联想"的舞台，降级为"被动检索"的候选。这正是遗忘的精确含义：**遗忘本质是降低参与后续计算的权重。** 一条记忆淡忘后仍可被精确匹配捞回，但不再主动"推荐"自己；直到某天你聊到高度相似的事，dense 把它从角落拉回，若再次被真正需要，strength 重新抬升，它又回到舞台。手指 episode 那 6 轮就是例子：编码时 salience 高、起步牢；之后几天每次聊到手指、绷带、换药都被召回、被强化。20 天无复习，以「切开引流」（初始 strength ≈ 2.55）为例 $2.55\times e^{-20/7}\approx 0.15$；但只要它还在那个共同激活的簇里、边还在传导能量，每次召回都是一次 `bounded_add` 把它从衰减中抬回来，就不会真的沉底。这就是"忘，是为了留住重要的"：系统不是平均主义地忘，而是有选择地忘——偏离日常重心、嵌入强关联簇、被真正需要时能拿到高 content_base 的记忆在时间中存活；和谁都连、和什么都不真正相关的，自然沉底。

钩子→ 状态在时间里筛出了一批"值得记住的"。但到此刻所有这些——salience、strength、resource、边——都只是零件。"此刻什么重要"的最终答案，是在一次具体 query 进来时，由这些零件在召回那一刻拼出来的。

## 四、召回 = 推荐：重要性在这一刻被算出来，而这个数同时就是学习信号

前三部分给了三样东西：节点（存什么）、边（怎么连）、时间状态（怎么筛）。现在看一次具体 query 进来，这些零件怎么被"读"成一个排序——并且**记住，这一步算出的 score，回答之后会原样变成 Part 2/3 里那个 $s$，去加固节点和边**。读和写共用这一个数，这是整套自稳定动力学的枢纽。

整条检索管线分五步：选种子 → 图扩散（graph_expand 发现 + RWR 微调）→ 合并 → noisy-OR 融合 → 注入。这里我们拿两个真实例子做讲解：**limboo（杀戮尖塔）** 和 **手指就医（鱼石脂·引流条）**。

**Step 1 · 种子选择：先点亮哪几盏灯。** 种子是扩散的出发点，三路并联各补短板（`core.py:747`）。**Dense（相似）**：query 向量和所有节点算 cos，> 0.675 入选，都不够就取 top-10 兜底——最稳的一路。**FTS（字面/稀有词）**：jieba 切词 → IDF ≥ 3.5 滤废词 → BM25 取 top-10，专补 dense 盲区的稀有专名；Dense 和 FTS 都命中时种子能量 ×1.3（封顶 1.5），纯 FTS 命中的限 5 条不让喧宾夺主。**BlackHole（高显著直连）**：salience > 0.8 且 cos > 0.60 的关键记忆不走任何边直接进种子池——为"意外但重要"的事开的绿色通道。limboo 的真实种子（"昨天在和 limboo 玩杀戮尖塔"）：

```
Dense+FTS  e=1.50  cos=0.966  昨天在和limboo玩杀戮尖塔…   ← limboo 稀有，双命中 ×1.3
Dense+FTS  e=1.50  cos=0.709  在和舍友玩杀戮尖塔2
Dense      e=1.00  cos=0.711  最近limboo的博客有什么新内容吗
FTS        e=1.00  cos=0.654  …存档覆盖…                  ← 纯靠字面"杀戮尖塔"被 FTS 捞回
BlackHole  direct  cos=0.668  slayspire那个单位是h…         ← 高 salience，绕过边直接进
```

每个种子初始能量还要除以 $\sqrt{1+\text{fan}}$——fan 大的 hub 即使当种子，初始能量也被打折。这是**第二道反 hub 防线**。

**Step 2 · 图扩散：graph_expand 是主力发现者，RWR 做同窗口微调。** 二者不是平行双通道，是主力 + 辅助的分工。**graph_expand（全图一跳，主力）**：拿 dense top-K 种子在整张大图上往外走一跳，不管时间远近——47 天前的记忆只要边够强照走不误；边信号用电导归一（$w/\sqrt{\text{出度}\cdot\text{入度}}$，反 hub），最终 score = $6.0\times\text{边信号}\times(0.25+\cos)\times(1+0.15\cdot\text{long})\times\text{resource}$。

**为什么 graph_expand 必须独立存在，而不是把 30 分钟窗口放大让 RWR 通吃？** 三个原因。第一，**计算成本**：RWR 要建 N×N 转移矩阵、做矩阵乘、迭代，微图 N≈60 瞬时完成，全图 N≈3000 每次 query 都跑差 2500 倍。第二，**hub 会吞掉所有能量**：RWR 是零和博弈，总能量固定、节点互抢；真实库有 fan 100–200 的 mega-hub，它们和谁都有弱边，窗口无限大时这些 hub 因入边多会吸走大部分能量，不管 query 是什么。30 分钟窗口是一道**语境界**——同 session 的边有意义（引流条→鱼石脂），跨天的弱边是噪音，放进来 ×36 一放大 hub 通吃一切。第三，也是最深层——**数学逻辑根本不同**：RWR 是能量再分配（零和竞争），适合稠密局部图（同情景内确实在竞争"谁更相关"）；graph_expand 是独立打分（互不影响），适合稀疏全局图（47 天前的一条记忆不该因为另一个簇被召回就被挤掉）。把小窗口放大，等于让 RWR 在稀疏全图上做零和竞争——不同情景的节点被迫互抢，hub 永远是赢家。

两步结果取并集（union + per-key max）。limboo（"昨天在和 limboo 玩杀戮尖塔"）这一跳拽回了什么：

| score | cos | dense 全库排名 | 来源 | 内容 |
|---:|---:|---:|---|---|
| **2.40** | 0.599 | 第 22 名 | Graph/1hop | 我是亡灵契约师 limboo是战士 |
| **1.48** | 0.598 | 第 24 名 | Graph/1hop | 和学校舍友联机玩 |
| **1.28** | 0.597 | 第 26 名 | Graph/1hop | 我玩机器人 limboo也玩机器人 |
| **1.27** | **0.436** | **第 1199 名** | Graph/1hop | 镜之边缘主角叫啥 |
| **0.99** | 0.601 | 第 21 名 | Graph/1hop | 和舍友俩机宝过了 a10 |

每条 cos 都远够不上 dense 种子门槛（0.675），全库排名 21–1199；但它们和好几条种子挂着 0.17–0.28 的共现边——这些边是过去几十次一起聊杀戮尖塔积累的。「镜之边缘」cos 仅 0.436、全库排 1199 名，纯相似度永远够不到——但边是活的，把它从一千名外拽到第 4。**这就是"重要性在召回时被算出来"的具体含义**：cos 是死的（「亡灵契约师」和「杀戮尖塔」字面不像、向量不像，cos 0.60 永远是 0.60），但边是活的（过去几十次共同提起，从 0 喂到 0.28），到召回这一刻把一条 cos 排名 22 的记忆推上榜首。

更极端的一个 case——**只拽回一条，这条不在 dense top-10 里，却拿了全场第一**。query：「但是拔出来引流条的时候出了好多血」（seq 7442）。dense top-10 全是和引流条/换药/麻药直接语义相关的记忆（cos 0.658–0.787）；「刚才白的是那种小点 鱼石脂是棕色的…」（seq 7276）cos=0.522，不在 dense top-10 里——它和"引流条出血"语义只有微弱重叠（都是手指 episode），纯相似度排不上号。但鱼石脂和引流条/麻药/绷带在手指就医 37 轮里被共同提起 **19 次**，每次共激活那条 STDP 边就加粗一阶，到 seq 7442 已粗到让 graph_expand 把它作为一跳邻居发现，电导归一后灌给它极高边信号：

| | cos | ripple（边信号） | score |
|---|---:|---:|---:|
| dense #1「不是换新的引流条吗」| 0.787 | — | **0.79** |
| **鱼石脂**（Graph/1hop）| **0.522** | **1.252** | **3.09** |

鱼石脂的 score 是 dense #1 的 **3.9 倍**。发现它（全图一跳）、给它打分（电导归一 × 边信号），全是 graph_expand 一个人干的——它离种子几小时远，不在 30 分钟微图里，RWR 根本看不到。这就是为什么 30 分钟窗口不能简单放大，而需要一套独立的、无时间限制的发现机制。

**Step 3 · 打分：noisy-OR——结算成那个统一的 score。** 通道跑完，每个候选进入统一打分（`core.py:966`）。两路证据按 noisy-OR 融合：

$$\text{content\_base} = \text{strong} + \text{weak}\cdot(1-\text{strong}),\quad \begin{cases}\text{ripple} = \text{current}\times 3\times\text{state} & (\text{扩散能量——动态、会学习})\\ \text{direct} = 0.5\times\cos & (\text{相似度——静态、不变化})\end{cases}$$

任何一路强都能把它拉高——扩散能量高但相似度低（鱼石脂 ripple=1.252, cos=0.522）照样冒头，相似度高但扩散能量低照样兜底，两路都强趋近 1.0。内容基底再乘三个来自前文状态变量的**乘法增益**（gain modulation，到这里那些"零件"才真正参与计算）：

$$\text{score} = \text{content\_base}\cdot\underbrace{(1+0.8\sigma)}_{\text{新颖}}\underbrace{(1+0.6\,\text{long})}_{\text{记得牢}}\underbrace{(1+0.5\,\text{edge})}_{\text{边强}}$$

增益都是 $1+x$ 形式——信号为 0 时增益为 1（不影响），信号高时 > 1；乘法意味着叠加而非覆盖（一个新颖又记得牢又边强的节点增益是 $1.8\times1.6\times1.5=4.32$ 倍）。最后乘一串惩罚：$\times\,\text{resource}\,\times\,\text{hop}\,\times\,\text{user}\,/\,(1+\text{fan})^{0.1}$——resource 刚用过的打折，hop 直连 1.0 > 一跳 0.86 > 两跳 0.62，user AI 独白节点 ×0.12，$/(1+\text{fan})^{0.1}$ 是**第五道反 hub 防线**。过阈值 **0.22** 才算"被激活"。

hub 在每一步都会重新冒头——种子端压了它扩散里涨回来，扩散压了它打分时涨回来。所以防线不是一道，是五道：

| # | 位置 | 公式 | 在哪一步 |
|---|---|---|---|
| 1 | 易激活度 | `state ÷ √(1+fan)` | Step 2（RWR state） |
| 2 | 种子初始能量 | `e₀ ÷ √(1+fan)` | Step 1 |
| 3 | 转移矩阵 | 每列只留最强 12 条边 | Step 2（RWR） |
| 4 | 一跳电导归一 | `w / √(出度·入度)` | Step 2（graph_expand） |
| 5 | 最终打分 | `÷ (1+fan)^0.1` | Step 3 |

limboo 例里「cs官方」fan 高达 282，就是在每道防线被逐层打折，最终 score 0.93——压在五条好的背景音之下。

**Step 4 · 注入：三个集合，各走各的路。** 检索产出的候选分进三个集合。**dense_items（左脑·精确回忆）**：纯 cos top-K，message 级匹配 → turn 映射，不经 RAR 管线、独立于图扩散——这是地板，永远兜底。**ripple_items（右脑·潜意识第一反应）**：RAR 全管线，soft_recall=True（放宽一档阈值，**宁可多给让 LLM 自己筛**），取 display_limit（最多 72 条），注入时只留 dense 没给过的——保证纯增量。**activation_items（左右脑融合）**：RAR 全管线，soft_recall=False，只取 **top 8**（`activate_limit`）；这 8 条在回答之后被写回——节点 strength↑、resource↓、recall+1，并和当前轮建 STDP 边。**只有这 8 条有资格参与强化**——这就是 Part 4 算出的 score 回流成 Part 2/3 学习信号的接口。

这里有一个非常核心的决策：**为什么 activation_items 是左右脑融合的 top-8，而不是只取右脑的 top-8？** 如果只强化右脑找出的联想，噪音会被自我放大：某话题里 dense 没覆盖的噪音节点碰巧被图扩散扫到 → 写回、加固边 → 下次更易被扫到 → 再加固……形成死亡螺旋，图烂成噪音 hub。全脑融合破解它：**dense 的事实永远以绝对相似度锚在候选池头部**，纯右脑挖出、没有 dense 兜底的噪音因相似度太低、content_base 不够，挤不进 top-8 → 进不了强化池 → 没资格 Hebbian 加固 → 那根细弱的边被 14 天衰减饿死。**Dense 不只是地板，还是 ripple 的免疫系统。** 注入 LLM 时分两块、都按时间排：左脑 dense top-10 + 右脑 ripple-only top-10，合计约 6000 字，LLM 这个"意识"做最终取舍。

## 五、闭环：今天的召回织成明天的图——以及它如何与你共同演化、自我净化

Part 4 是一条 query 从进到出的完整旅程。但旅程的终点不是注入 LLM——回答之后还有一步**写回**，把这次召回的成果织进图里，成为下一次召回的路。回答之后，系统拿出刚才的 `activation_items`（左右脑融合 top-8），对其中每条旧记忆 $A$ 和当前轮 $T$ 落三种边（$A\to T$ causal $1.0\cdot s_A$；$T\to A$ acausal $0.35\cdot s_A$；$A\leftrightarrow B$ 共激活 $\sqrt{s_A s_B}$），同时更新 $A$ 的节点状态（$\text{strength}\leftarrow\text{bounded\_add}(\text{衰减后}, 0.18\cdot s_A, 3.0)$，resource 消耗、recall+1）。**一次完整回合 = 检索 + 回答 + 学习**：读和写是同一套规则的两种调用——读时扩散能量在边上走，写时增量在边上落。写回是异步的且滞后一轮（当前轮写下的边下一轮才用得上）——这就是**因果约束**：你不能用"未来"的边影响"过去"的检索。

但闭环比"边会长大"更深一层，而且这一层是 Akasha 真正区别于一切快照式记忆系统的地方。普通向量库的检索是**只读**的：召回什么，不改变写入什么（写入就是原始对话），一次无关召回不过是几个被浪费的 token、无后果。Akasha 不是——**召回会塑造回答，回答会变成新记忆的内容，新记忆又参与未来的召回**：

$$\text{召回}\;R_t\;\longrightarrow\;\text{LLM 回答}\;A_t(\text{被 }R_t\text{ 塑造})\;\longrightarrow\;\text{新节点 }N_t(\text{内容带着 }R_t\text{ 的影子})\;\longrightarrow\;\text{参与 }R_{t+1}\;\longrightarrow\cdots$$

这是一个**内容级的闭环**，比"边结构的闭环"强一个量级——召回直接决定了"这段对话是关于什么的"，从而决定了沉淀下来的记忆是什么。那条看似"噪音"的联想召回，在闭环里不是被动上下文，而是**给对话的一个 prompt**：它把 limboo 的"亡灵契约师/舍友"带出来，LLM 顺着聊，你回应，这段关系被进一步坐实成新记忆、边被加固。于是系统不是在**存**你的世界，而是在**主动组织、并通过对话反复强化它对你世界的模型**。而这只在长时程里显现——会话越长，共现越积越多、边越练越粗、涌现越频繁，"背景被它慢慢完全掌握"。快照向量库没有这个雪球：写入即定型，不会越来越懂你。

闭环放大它召回的一切——signal 放大，noise 也放大，所以必须有一个反向力，否则一次坏召回会顺着同一个雪球滚成死亡螺旋。这个反向力就是**租金原理的自净化**：一条只靠时序巧合连上、却无相似无相关的边，因为不会被再次共激活，就交不起租——在 14 天衰减下被饿到近静音。结构上反复加固的相关边有效权逼近上限且新鲜，一次性巧合、十几天没再碰的无关边在 14 天 τ 下衰减到近零，两者可以差出两三个数量级。坏的不需要谁专门去清理，它自己枯掉；好的也不需要谁专门去奖励，它在使用中自我加固。这正是开篇那条原理的两个方向在闭环里同时兑现。

于是"右脑宁可多给（soft_recall）让 LLM 筛"这个看似冒险的设计，其实站在四重锚定之上：**dense 地板**保证准确性（任务/事实的精确命中永远兜底，和右脑漂不漂无关）；**有界增长 + 衰减**保证投机错了不会爆、会自清；**五道反 hub** 压住靠虚假复现活着的 mega-hub（这类不会因衰减消失，需要专门压）；**LLM 充当前额叶**做最终取舍，就此检索到的内容被 llm 自然过滤，生成被背景音影响到内容，从而真正让回答是了解完整上下文的。这也是为什么系统能安全地联想：滥进来的无关项，要么被 LLM 跳过，要么因不被复现而枯死，伤不到根。需要补一句的是——衰减只清"无相关无复现"的噪音，清不掉"你好/天气"这种在无关语境里反复出现、边一直被刷新的 hub；所以衰减和反 hub 是互补的两半，合起来才是闭环的免疫系统。

现在回头看开篇那条原理——每一层各给了什么：

| 层 | 给什么 | 在租金原理里的角色 |
|---|---|---|
| 编码（一） | embedding + salience/strength/resource 中性起点 | 不预判：先不收租，给一个公平起点 |
| 边（二） | Hebbian 关联图、用进废退 | 关系的租金：共激活续租，不用则衰减 |
| 时间（三） | 三条衰减 + 负相关实证 | 状态的租金：被需要才留，次数≠重要 |
| 召回（四） | noisy-OR + 五道反 hub | 一次续租结算：算出 score，既给 LLM 又回流成学习信号 |
| 闭环（五） | 内容级共同演化 + 自净化 | 复利与免疫：好的滚强，坏的枯死 |

没有哪一层单独是"重要性"。编码给中性起点，边给关联结构，时间筛掉噪音，召回那刻按语境现算，闭环让结果反哺自身——重要性是在每一步被逐步"挣"出来、又被持续"续租"出来的，不是在任何一步被"判"出来的。
