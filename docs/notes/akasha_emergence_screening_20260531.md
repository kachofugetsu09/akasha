- 生成时间：2026-05-31

- 原始素材：GPT 分类的 100 个 case（50 好 / 50 差），见 `akasha_live_case_collection_20260531.md`
- 本文只看 **live**（query_log 里线上真实落库的激活/涟漪），不依赖 replay 重建。replay 只用来判断"上限"，见末节。
- 复筛口径：**我们要的是"需要的、类似人类右脑推理的涌现"，不是噪音数量的多少。** 一个 case 好不好，不取决于图扩散激活了多少节点、ripple 有多大，而取决于扩散**有没有落在当前真正需要的那条联想链上**。

---

## 一、核心发现：涌现质量与扩散量正交

我对 100 个 case 算了几个客观量（Activation top5）：平均 fan（节点度，越大越是 hub）、最大 fan、graph-only 命中数（top5 里走图扩散、且 dense 没单独捞到的条目数）。

| 指标 | 好 case 均值 | 差 case 均值 |
|---|---:|---:|
| top5 平均 fan | 93 | 96 |
| top5 最大 fan | 175 | 168 |
| top5 graph-only 命中数 | 3.6 | 4.0 |

**结论：好坏两组在"扩散量级"上几乎不可分**——差 case 的图扩散甚至更活跃（graph-only 更多）。这正面印证了"不能用噪音/扩散数量评价好坏"。

> 决定一个 case 是不是"右脑涌现"的，是**语义落点**：
> - **好（真涌现）**：query 本身很短/很间接/几乎没有字面线索，图扩散却把当前真正需要的那段 episode 或概念簇拽了出来——这是 dense 相似度单独做不到的"联想跳跃"。
> - **差（噪音）**：扩散落到了与当前话题无关的 **mega-hub**（天气 / "说说看我都有什么仓库" / "你好啊" / "你真是claude吗" / 晚安关系簇）或**跨域串线**上。激活值再高也只是 hub 的引力，不是推理。

## 二、判定标尺

**好 case 分三档：**
- **A｜真涌现**：query 几乎不含答案的字面词，靠图扩散跳到所需 episode。dense 单独给不出。← 最有价值，是这套系统存在的理由。
- **B｜有效但偏共现/左脑可达**：召回正确，但要么 query 本身就带强 anchor（dense 也能捞到），要么只是同一 episode 紧密共现，谈不上"跳跃"。
- **降级**：GPT 标了好，但 live 实际被 hub 稀释 / 漂到邻近但错误的簇，不该算正面样本。

**差 case 按失败模式分四类：**
- **N1 无锚漂移**：query 没有语义锚点（"现在呢""讲""我现在状态怎么样"），扩散落进 mega-hub。
- **N2 跨域串线**：一句话里塞多个话题，扩散分叉到强势的那个域。
- **N3 稀有短词/私梗压制**：稀有实体（2236、🦌）被高频问候/私梗簇盖掉。
- **N4 实时/精确事实（"天花板"类）**：召回其实连贯且相关，但答案必须实时查/重算（新闻、额度、花费、日期、传感器、文件），图召回只能当背景，不能直接信。这类**不是噪音**，单独列。

## 五、replay 对照与"上限"判断

依据原报告的安全 replay 复跑（候选库 `/tmp/akasha-safe-full-replay-FMeV70/akasha.db`）：

- **Dense 100% 可复现**（3327/3327 全量一致）→ dense 是稳定地板，也是"涌现"的对照基线。
- **Activation/Ripple 大面积变化**（全量 content 完全一致仅 ~16%，100 case 里 17 变差 / 5 变好 / 64 中性）→ 图状态是**路径依赖**的，不是等价产物；replay 并没有系统性地"变好"。
- 因此 **replay 不构成更高的上限**：它说明图扩散结果对历史轨迹敏感、可复现性差，而不是"还有很多召回潜力没释放"。真正的上限信号是：**好坏两组扩散量相同（见第一节），所以提升空间在"语义路由/抑制 hub"，而不是"扩散得更多"。**

## 六、给算法的启示（基于复筛）

1. **hub 抑制是第一优先**：天气、"什么仓库"、"你真是claude"、晚安关系簇、"分析我喜欢什么视频"这些超高 fan 节点反复污染无锚 query。按 fan/IDF 做激活惩罚或度归一化，收益最大。
2. **无锚 query 应短路图扩散**：长度极短、无实体（"讲""现在呢""晚安"）或纯问候时，降权图召回、回退 dense 或上一轮上下文，避免 hub 主导。
3. **实时/精确意图要门控（N4）**：识别"价格/额度/日期/天气/传感器/新闻/文件 grep"意图时，把图召回标记为"背景参考、需实时核验"，不进决策。
4. **affect 簇限流**：关系/情绪簇召回有用但极易过激活，应设注入配额（用户已点名）。
5. **真涌现样本稀少且珍贵**：A 级（seq 7372 / 6922 / 7374 / 7340 / 1783 这类）应作为评测金标准——短 query + 无字面线索 + 图扩散跳到所需 episode + dense 给不出。把这五条沉淀成回归用例，比追求"召回更多"更能衡量右脑能力。
